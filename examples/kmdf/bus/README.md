# NTL KMDF bus and PDO sample

This sample shows a root-enumerated KMDF bus FDO and a child function driver
communicating through a typed NTL KMDF query interface. The bus driver:

- configures a typed dynamic `child_list`;
- receives `ntl::kmdf::pdo_init` in the child-create callback;
- assigns device, instance, hardware, compatible, and localized text data;
- reports an empty logical resource configuration for the virtual PDO through
  typed resource-requirements and boot-resource callbacks;
- creates an `ntl::kmdf::pdo` with managed C++ context storage;
- registers typed eject, lock, wake, and reported-missing PDO callbacks;
- exposes a reference-counted driver-defined interface from the PDO; and
- validates the PDO's identification and parent relationship.

The child function driver obtains that contract with
`device::try_query_interface()`, calls the bus-owned functions, and publishes a
user-mode device interface. The app opens the bus management interface and
validates `plug in -> query -> mark missing -> plug in -> eject`, including
resource callback counts, SetupAPI arrival, and removal of the child function
stack.

## Visual Studio

Open `crtsys_kmdf_bus_ntl_sample_vs.sln`, restore the `crtsys` package, and
build the x64 Debug or Release configuration. The solution contains the bus
driver, child function driver, and validation app. During repository
development, `examples/Directory.Build.props` selects the local package
generated by the repository.

Install the root-enumerated bus from an elevated PowerShell prompt:

```powershell
.\install.ps1 -PackageDirectory .\x64\Debug
```

Then run:

```powershell
.\x64\Debug\crtsys_kmdf_bus_ntl_sample_app.exe
```

Remove it with:

```powershell
.\remove.ps1
```

## CMake

```powershell
cmake -S examples\kmdf\bus `
      -B artifacts\examples\kmdf-bus -A x64
cmake --build artifacts\examples\kmdf-bus --config Debug
```
